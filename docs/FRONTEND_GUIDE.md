# 프론트엔드 가이드

## 기술 원칙

- HTML, CSS, Vanilla JavaScript만 사용한다.
- 기존 디자인 시스템과 DOM 구조를 우선한다.
- 공통 로직은 중복하지 않는다.
- 페이지별 파일 책임을 분리한다.

## 내비게이션

### 메인 메뉴

- 대시보드
- 아카이브
- 강의실

### 숏컷

- 최근 문서
- 즐겨찾기 또는 고정 문서
- 휴지통

## 인증 화면

최소 화면:

- 회원가입
- 로그인
- 인증 오류 메시지
- 로그아웃 버튼

로그인 성공 시 대시보드로 이동한다.

## 보호 페이지

보호 대상:

- 대시보드
- 아카이브
- 원고 상세
- 에디터
- 강의실

페이지 진입 시 현재 사용자 API를 확인하고, 401이면 로그인 페이지로 이동한다.

## `api.js` 권장 책임

```js
// 개념 예시
async function apiFetch(path, options = {}) {
  // API 기본 주소 결합
  // 인증 정보 추가
  // 공통 오류 처리
  // JSON 응답 파싱
}
```

## 대시보드 구성

- 사용자 인사 또는 계정 정보
- 최근 원고
- 원고 수
- 최근 활동
- 즐겨찾기
- 강의실 요약

## 에디터 구성

- 원고 제목
- 본문 편집 영역
- 자동 저장 상태
- 저장 버튼
- 되돌리기
- 버전 히스토리
- 보조 패널

상단 상태 표시줄은 중앙 작업 영역까지만 차지하고, 오른쪽 보조 사이드바는 화면 위쪽부터 독립적으로 시작하는 레이아웃을 유지한다.

## 테마

사이드바 색상을 자유 선택하는 방식보다, 사전에 정의된 테마 6개를 선택하는 방향을 우선한다.

최소 후보:

- 블랙
- 아이보리 화이트
- 딥 블루
- 슬레이트
- 웜 그레이
- 다크 그린

실제 구현 시 기존 CSS 변수와 디자인을 먼저 확인한다.
