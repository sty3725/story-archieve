# 개발 로드맵

## Milestone 0 — 저장소 기준선 확립

완료 조건:

- 실제 실행 명령 확인
- 백엔드와 프론트엔드 시작 방법 확인
- 환경변수 목록 확인
- 데이터베이스 연결 확인
- 현재 테스트 또는 수동 검증 절차 기록
- 문서와 실제 구조의 차이 기록

## Milestone 1 — 인증 흐름 완성

작업:

1. 회원가입 DB 저장 검증
2. 비밀번호 해시 검증
3. 로그인 API 구현 또는 수정
4. 인증 정보 발급
5. 현재 사용자 API
6. 대시보드 보호
7. 로그인 폼 연결
8. 로그인 성공 후 이동
9. 로그아웃
10. 오류 메시지

완료 조건:

- 신규 사용자 등록 가능
- 등록 사용자 로그인 가능
- 비밀번호 오류 시 401
- 비로그인 상태에서 대시보드 API 접근 불가
- 로그인 후 대시보드 표시
- 로그아웃 후 접근 차단

## Milestone 2 — 대시보드 최소 기능

작업:

- 사용자 정보
- 원고 수
- 최근 원고
- 즐겨찾기
- 빈 상태

완료 조건:

- 로그인 사용자 데이터만 표시
- 데이터가 없을 때 깨지지 않음
- 최근 수정 순서가 정확함

## Milestone 3 — 아카이브와 원고 CRUD

작업:

- 목록
- 생성
- 상세
- 수정
- 삭제 또는 휴지통
- 검색
- 즐겨찾기

완료 조건:

- 사용자별 소유권 검사
- CRUD 성공
- 삭제 정책 일관성
- 프론트와 API 연결

## Milestone 4 — 에디터와 저장

작업:

- 원고 본문 편집
- 저장
- 자동 저장
- 저장 상태 표시
- 동시 요청 방지
- 실패 복구 메시지

완료 조건:

- 새로고침 후 내용 유지
- 저장 실패를 사용자에게 표시
- 불필요한 저장 요청 억제

## Milestone 5 — 버전 이력

작업:

- 버전 생성
- 목록
- 미리보기
- 복원
- 변경 메모

## Milestone 6 — 강의실 최소 기능

실제 요구사항과 모델을 확정한 뒤 진행한다.

## 각 작업의 공통 완료 기준

- 변경 파일 명시
- 실행 가능
- API 동작 확인
- 오류 처리
- 인증 및 소유권 확인
- 문서 갱신
