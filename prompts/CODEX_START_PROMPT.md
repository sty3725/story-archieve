# Codex 첫 작업 프롬프트

아래 내용을 Codex의 첫 요청으로 사용한다.

---

이 저장소는 FastAPI + PostgreSQL + HTML/CSS/Vanilla JavaScript로 만든 Story Archive 프로젝트다.

먼저 다음 문서를 순서대로 읽어라.

1. `AGENTS.md`
2. `docs/PROJECT_OVERVIEW.md`
3. `docs/ARCHITECTURE.md`
4. `docs/AUTH_FLOW.md`
5. `docs/DEVELOPMENT_STATUS.md`
6. `docs/ROADMAP.md`

그 다음 실제 저장소 전체를 검사하고, 문서의 예상 구조와 실제 코드가 어떻게 다른지 확인하라.

이번 작업 범위는 다음과 같다.

## 목표

회원가입된 사용자가 로그인하고, 인증 성공 후 대시보드로 이동하며, 비로그인 사용자는 대시보드와 보호 API에 접근할 수 없도록 만든다.

## 반드시 먼저 확인할 것

- 회원가입 시 실제 PostgreSQL에 commit 되는지
- 비밀번호가 해시로 저장되는지
- 기존 로그인 라우터 또는 인증 유틸이 있는지
- JWT, 쿠키, 세션 중 현재 사용 중인 방식
- 대시보드 API와 페이지의 현재 보호 방식
- 프론트엔드 API 호출 구조
- CORS, 정적 파일, 환경변수 설정
- 기존 테스트 또는 실행 명령

## 구현 범위

- 로그인 API 확인 및 필요한 수정
- 현재 로그인 사용자 조회 API
- 인증 의존성 또는 공통 인증 검증
- 대시보드 API 보호
- 로그인 폼과 API 연결
- 로그인 성공 후 대시보드 이동
- 인증 실패 시 로그인 페이지 이동
- 로그아웃 처리

## 제한

- 기존 구조를 임의로 대규모 변경하지 마라.
- 프론트엔드 프레임워크를 추가하지 마라.
- 문서보다 실제 코드를 우선하라.
- 비밀키나 DB 주소를 하드코딩하지 마라.
- 데이터베이스 모델 변경이 필요하지 않으면 변경하지 마라.
- 관련 없는 기능은 수정하지 마라.
